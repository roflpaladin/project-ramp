## 1. What Ramp is

Ramp is a shared workspace where a seller and a buyer commit to a plan and act on it together. Two people, one plan, one next move.

## 2. Core principle: never make them guess

Every surface answers four questions before anyone has to ask:

1. Where am I in the plan?
2. What's already done?
3. What's mine versus theirs?
4. What is my one next move?

Calm does not come from emptiness. It comes from **orientation**. A sparse screen that leaves you wondering is a failure; a dense screen that answers everything instantly is a success.

**The feeling.** Finishing at 4pm because someone helped you with the thing you were dreading. Getting in the car, golden hour through the windscreen, smooth jazz on the radio. Warm, unhurried, entrusted, in good hands. Not clinical. Not urgent. Not clever.

- `MUST` — If a screen cannot answer all four questions, the screen is incomplete.
- `MUST` — Ramp is a guide, not a dashboard. It points; it does not merely report.

## 3. Colour

`MUST` — Define every colour as a CSS variable or design token. Never scatter raw hex through components.

### 3.1 The loudness hierarchy

`MUST` — Every colour in Ramp does exactly one of four jobs. If a colour is not doing one of these, use a neutral.

- **Neutrals** — the room. ~90% of every screen. Should feel invisible.
- **Sides** *whisper* — who owns the move (identity).
- **States** *speak* — what's happening (status).
- **Signal** *shouts* — the next action, and nothing else.

`MUST` — Never introduce a fifth loud colour.

### 3.2 Signal — golden hour

Signal is a warm amber. It is the sunset through the windscreen, the ramp rising, the hand pointing at your next move. It is warm on purpose: the action moment should feel like relief, not like an alarm.

- `MUST` — Signal is used **only** for: the primary action, the current/live step, and the ramp gesture.
- `MUST` — Never use Signal for headings, body text, decoration, backgrounds, or "to add colour."
- `MUST` — **One Signal element per decision scope.** A decision scope is a page, a modal, a drawer, or a single card. Two Signals in one scope means neither reads as the action.
- `MUST` — Signal never encodes status. Status is the States palette.

### 3.3 Waiting is quiet

"Waiting on the other side" is deliberately **not** a colour. It renders in Slate.

Waiting is the most common state in a deal and the least urgent. Giving it a loud colour trains people to ignore loud colours. Only two states are allowed to speak: done, and at risk.

### 3.4 Tokens

```css

  --mist:         #F1F0EB;  /* raised surface, fills */

  --slate:        #6B6E76;  /* secondary text, meta, WAITING state */
  --ink:          #17181C;  /* primary text */

  /* Signal — action only */
  --signal:       #A85B12;  /* 5.0:1 on white — AA pass */

  --signal-hover: #8F4C0D;
```

- `MUST` — Every surface ships both themes. No light-only or dark-only components.
- `MUST` — Never hardcode `#FFFFFF` or `#000000`. Use `--paper` / `--onyx`.

### 3.5 Contrast floor

- `MUST` — Body and label text ≥ 4.5:1 against its background. Large text (≥24px or ≥19px bold) and UI borders/icons ≥ 3:1.
- `MUST` — **Status is never colour-only.** Every state renders as dot + text label, always. A colourblind user must lose zero information.

## 4. Buyer branding (whitelabel)

Ramp scrapes the buyer's domain and themes their portal. This is the most brand-critical surface we ship, so the boundary is hard.

**Ramp owns the chrome. The buyer gets an accent.**

- `MUST` — Ramp's neutrals, typography, spacing, components, and Signal are fixed. The buyer's brand never replaces them.
- `MUST` — Buyer logo appears only in the header identity strip, max 24px tall, on a neutral background. Never in the body, never as a watermark, never tiled.
- `MUST` — The buyer accent colour is permitted in exactly two places: the identity strip rule/underline, and the browser favicon + tab title. Nowhere else.
- `MUST` — The buyer accent never appears on a button, never replaces Signal, never encodes status.
- `MUST` — If the scraped accent fails 3:1 against its background, discard it and fall back to `--ink`. Never render an unreadable brand.
- `MUST` — On scrape failure or timeout, render generic Ramp chrome. A missing logo is never an error state, an empty box, or a broken image.
- `MUST` — Branding is cached per `target_domain`. Workspace A's branding must never bleed into Workspace B.

**Why:** the buyer should feel *hosted*, not *impersonated*. Their logo says "this is about you." Ramp's chrome says "and someone reliable is running it."

## 5. Typography

- `MUST` — One family: **Geist**. Stack `"Geist","Inter",system-ui,-apple-system,sans-serif`.
- `MUST` — Data, numbers, IDs, timestamps, currency, CRM/integration references: **Geist Mono**. Stack `"Geist Mono","JetBrains Mono",ui-monospace,monospace`.
- `MUST` — Weights 400, 500, 600 only. No new fonts, no 700+. Let weight carry hierarchy.
- `MUST` — Sentence case everywhere. Never Title Case. Never ALL CAPS — including badges, nav, and table headers.

### Scale

| Token | Size / line | Weight | Tracking | Use |
|---|---|---|---|---|
| `display` | 40 / 44 | 600 | -0.03em | Marketing hero only |
| `h1` | 32 / 38 | 600 | -0.02em | Page title |
| `h2` | 24 / 30 | 600 | -0.015em | Section |
| `h3` | 18 / 24 | 600 | 0 | Card title, milestone name |
| `body-lg` | 16 / 24 | 400 | 0 | Portal body, buyer-facing prose |
| `body` | 14 / 21 | 400 | 0 | App default |
| `meta` | 13 / 18 | 400 | 0 | Secondary, Slate |
| `micro` | 12 / 16 | 500 | 0.01em | Badges, table headers, mono data |

`SHOULD` — Prose line length caps at 68 characters.

## 6. Space, shape, depth

- `MUST` — 4px base grid. Spacing scale: `2, 4, 8, 12, 16, 24, 32, 48, 64, 96`. No arbitrary values.
- `MUST` — Radius scale: `6px` inputs and badges · `10px` buttons · `12px` cards · `20px` sheets and modals · `999px` pills and avatars.
- `MUST` — Borders are `1px solid var(--line)`. Hairline only.
- `MUST` — No shadows on in-flow elements. No gradients anywhere except the ramp gesture.
- `SHOULD` — Overlays (modal, popover, toast) may use exactly one shadow: `0 8px 32px rgba(23,24,28,0.10)`.

**Density.** Same tokens, two postures.

- Seller app: `body` 14px, row height 44px, spacing step 12–16.
- Buyer portal: `body-lg` 16px, row height 56px, spacing step 16–24, roughly 40% fewer controls per screen.

`MUST` — Never fork the component library for the buyer. Same components, different density props.

## 7. Motion

Motion is how "someone is helping you" becomes physical. It should feel like a door being held open — never like a spring.

- `MUST` — Durations: `120ms` micro (hover, focus) · `200ms` standard (state change, reveal) · `320ms` surface (modal, drawer, page).
- `MUST` — Easing: `cubic-bezier(0.2, 0, 0, 1)`. No bounce, no elastic, no overshoot.
- `MUST` — Respect `prefers-reduced-motion`: fall back to opacity-only, ≤120ms. Never remove the transition entirely — an instant jump is disorienting.
- `SHOULD` — When a step completes, the Signal moves to the next step rather than fading out and reappearing. The pointing finger travels.
- `MUST` — Nothing loops, pulses, or animates on idle. Motion is always a response to something.

## 8. Components

### Button

`MUST` — Height 40px (36px in dense seller tables). Radius 10px. Weight 500. Horizontal padding 16px.

- **Primary** — `--signal` fill, `--signal-fg` text. Hover `--signal-hover`, pressed `--signal-press`. Disabled: `--mist` fill + `--slate` text, no amber.
- **Secondary** — `--paper` fill, `1px solid var(--line)`, `--ink` text. Hover `--mist`.
- **Tertiary** — no fill, no border, `--ink` text. Hover `--mist`.
- **Destructive** — secondary shell with `--state-risk` text. Only inside a confirmation dialog, never on a primary surface.

- `MUST` — Exactly one primary per decision scope.
- `MUST` — Focus ring: `2px solid var(--signal)` at `2px` offset, always visible on keyboard focus.
- `MUST` — Loading state replaces the label with a spinner and keeps the button's exact width. No layout shift.

### Status badge

- `MUST` — 6px dot + `micro` label. `--wash-done` / `--wash-risk` background for done and risk; `--mist` for waiting. Radius 999px, padding 4/8.
- `MUST` — Always has a text label. Never a bare dot.

### Milestone row

- `MUST` — 3px left rail in the owning side's marker colour. Status dot + badge. Title in `h3`. Owner and due date in Geist Mono `micro`, Slate.
- `MUST` — The live step gets a `--signal-wash` background and the ramp gesture as its left rail. Only one live step exists per plan.
- `MUST` — Side tint and state colour compose; they never override each other. The rail says *who*, the badge says *what*. If they ever conflict visually, the rail loses saturation — never the badge.

### Card

`MUST` — `--paper` on `--mist` (or `--mist` on `--paper` in dark). 12px radius, hairline border, 24px padding. No shadow.

### Input

- `MUST` — Height 40px, 6px radius, `1px solid var(--line)`, `--paper` fill. Focus: border `--signal` + 2px ring. Error: border `--state-risk` + message below in `meta`.
- `MUST` — Labels sit above the field, always visible. Never placeholder-as-label.

### Empty state

`MUST` — One sentence naming what goes here, plus the primary action. Never an illustration, never "Nothing to see here."

## 9. The ramp mark

- `MUST` — Wordmark is `ramp`, lowercase. Letters stay `--ink` (light) or `--paper` (dark). Only the underline carries Signal.
- `MUST` — Signature gesture: a rising plane inclining left-to-right toward action — `clip-path: polygon(0 100%, 100% 0, 100% 100%)` or equivalent. Same angle everywhere.
- `MUST` — Reuse the gesture as: logo underline, app icon, section marker, and the fill behind the live step. Nowhere else.
- `MUST` — Never recolour the wordmark, stretch it, outline it, add gradients or shadows, or tilt the whole mark.

---

## 10. Voice

- `MUST` — Say the next mutual move and nothing else. Active voice, sentence case, no hype, no emoji.
- `MUST` — Name what both sides do next: *"Your buyer approved the plan. Set the next step."* · *"Two steps left before go-live."* · *"Nothing pending on your side. Their move."*
- `MUST` — Buttons name the action and keep the same word through the entire flow. "Confirm rollout date", not "Submit".
- `MUST` — Errors state what happened and what to do, in the product's voice. Never apologise, never vague, never a raw exception string.
- `MUST` — Banned: leverage, synergy, supercharge, seamless, unlock, empower, effortless, revolutionise — and `!` in system copy.

**Bad news is stated plainly, with the fix attached.** No banners, no alarm, no escalation styling. A blocked step reads: what's blocked, why, and the button that unblocks it — in place, in the row where it lives.

## 11. Quality floor

- `MUST` — Responsive to 360px width.
- `MUST` — Visible keyboard focus on every interactive element.
- `MUST` — Full keyboard operability: no mouse-only interactions.
- `MUST` — `prefers-reduced-motion` respected.
- `MUST` — Both themes ship.
- `MUST` — Contrast floor met (§3.5).
- `MUST` — No layout shift on load or state change.

---

## 12. When in doubt

1. Does the screen answer *where am I, what's done, what's mine, what's next*? If not, fix that before anything else.
2. Is this colour doing identity, status, or action? If not, make it neutral.
3. Is there exactly one Signal in this scope?
4. Would this feel like being helped, or like being managed?

If a rule doesn't cover your case, default to the neutral, the quieter option, and the smaller step — then add the rule to this page.
